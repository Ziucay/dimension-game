Переход в другой мир работает по нажатию кнопки “F” (сделано временно). Если при переходе мы хотим поменять префаб, то мы накладываем на него скрипты. Скриптами управляет ‘AlternativeWorldManager’. По умолчанию отображается первый мир. 
Добавлять можно несколько скриптов на один префаб. 
В полях скрипта нужно добавить значения public переменных.

Типы скриптов:

- InvisiblePrefab. Префаб полностью исчезает.
- В поле ‘isVisibleInFirstWorld’ ставим true, чтобы этот префаб был виден только в первом мире, false - только во втором.
- Пример. Блок, существующий только в каком-то одном мире.

- ColliderPrefab. Префаб становится неактивным: игрок может проходить сквозь него.
- В поле ‘isCollidesInFirstWorld’ ставим true, чтобы этот префаб имел коллайдер только в первом мире, false - только во втором.
- Пример. В первом мире видим закрытую дверь. Во втором миру также видим эту дверь, но теперь можем пройти сквозь нее, так как это просто “проекция” префаба из первого мира.

- SecondSpriteChangePrefab. Префаб меняет свой спрайт. Имеет второй спрайт в альтернативном мире.
- В поле ‘secondSprite’ устанавливаем спрайт, на который должен поменяться префаб, при перемещении во второй мир. В первом мире спрайтом будет являться текстурка, выставленная изначально у префаба.
- Пример. В первом мире видим дорогу с травой, во втором - дорогу покрытую снегом.

- SpriteChangePrefab. Префаб меняет свой спрайт. Логика такая же, как у предыдущего ‘SecondSpriteChangePrefab’.  Для каждого из миров префаб имеет свой собственный спрайт. Также есть изначальный спрайт, который отображается, если ни разу не нажимали на клавишу перемещения в другой мир. Он может понадобится для отображения “нетронутости” префаба от использования перемещения в альтернативный мир.

Эти базовые спрайты можно комбинировать, чтобы создавать определенное поведение у префаба, которое мы хотим при перемещении в другой мир.
Пример. Добавим два спрайта ‘ColliderPrefab’ и ‘SecondSpriteChangePrefab’ на объект. В первом мире он выглядит как обычный объект. Во втором мире будет отображаться только его обводка по краям (другой спрайт) и отключен коллайдер (игрок сможет проходить сквозь него). Таким образом, во втором мире будет отображаться только силуэт объекта, и игрок будет знать, что нужно переключиться в первый, чтобы взаимодействовать с ним.
